# プロジェクト概要

## 製品名
**WBS Manager** - ローカルファースト WBS管理ツール

## 概要
共有フォルダを使用した、10人規模のチーム向けWBS（Work Breakdown Structure）管理Webアプリ。
単一HTMLファイルで動作し、インストール不要。

## 背景・課題

### 解決したい課題
- 既存タスク管理アプリはフラット構造で、プロジェクト全体の階層的な進捗把握が困難
- Excelガントチャートは更新が煩雑で、リアルタイム共有が難しい
- 外部SaaS（MS Project、Asana等）は会社のセキュリティポリシーで使用不可

### ターゲットユーザー
- 10人以内のチーム
- WBSの作成・更新は主にチームリードが担当
- 他メンバーは閲覧・自分の担当分の進捗更新がメイン

## 制約条件

| 項目 | 制約 |
|------|------|
| 外部サービス | 利用不可 |
| サーバーサイド | 構築不可（Python/Node.js環境なし） |
| 実行環境 | ローカルHTMLファイル |
| データ保存 | 共有フォルダ（SharePointをネットワークドライブとしてマウント） |
| ブラウザ | Chrome / Edge のみ（IE, Firefox 非対応） |

## 設計思想

### WBS管理とタスク管理の役割分担

| 観点 | タスク管理 | WBS管理（本アプリ） |
|------|-----------|---------------------|
| 構造 | フラット | 階層構造 |
| 時間軸 | 短期・日常 | 中長期・プロジェクト全体 |
| 主な関心 | 「誰が何をやるか」 | 「全体のどこまで進んだか」 |
| 依存関係 | 基本なし | 重要（先行・後続） |
| 進捗管理 | 完了/未完了 | %単位 + 子要素からの集計 |

### 粒度ガイドライン

| レベル | 粒度 | 例 |
|--------|------|-----|
| Level 1 | フェーズ | 設計、開発、テスト |
| Level 2 | 成果物・マイルストーン（3〜5日） | 要件定義書、API実装 |
| Level 3 | サブ成果物（任意） | 画面A、画面B |

**WBSは「何を作るか」、タスクは「どう作るか」**

### 突発作業の扱い
「運用・サポート」という特殊なWBS枠（`type: support`）を設け、計画外の作業（問い合わせ対応等）を管理。

## 技術的な限界

HTMLローカル実行の制約により、以下は実現不可：

| できないこと | 理由 |
|-------------|------|
| リアルタイム同期 | WebSocketサーバーが必要 |
| ファイル変更の即時検知 | ファイル監視APIがブラウザにない |
| 認証・権限管理 | サーバーサイドが必要 |
| 自動バックアップ | スケジュール実行不可 |
| 同時編集のマージ | CRDT等の実装が複雑 |

### 運用でカバーする点
- 同じWBSを同時編集しない（声かけ or チャット連絡）
- 定期的に手動バックアップ（フォルダごとコピー）
- 60秒間隔の自動リロードで他メンバーの変更を反映
